# PRD: Agile Metrics Hub

**作成者**: Ryusei | **日付**: 2026-04-06 | **ステータス**: ドラフト
**アイデアID**: 3598 | **ドメイン**: プロジェクト管理 | **難易度**: Medium

## 問題定義

プロジェクト管理ツール（ClickUp, Jira, Linear, Asana等）のアジャイルメトリクス機能（バーンダウン、ベロシティ、累積フロー、リードタイム分析）は、多くの場合**有料プラン限定**で提供される。小規模チームやスタートアップにとって、各ツールの上位プランに課金するのは負担が大きい。さらに、複数ツールを併用するチームでは、メトリクスが分散して横断的な分析ができない。

Agile Metrics Hub は、PMツールのAPIキーを設定するだけで、ツール横断のアジャイルメトリクスダッシュボードを提供する。

## 目標と成功指標

| 目標 | 指標 | ターゲット |
|------|------|----------|
| PMツール連携の容易さ | APIキー設定からダッシュボード表示までの時間 | 5分以内 |
| メトリクスの正確性 | バーンダウン/ベロシティのデータ整合性 | PMツール上のデータと一致率 95%+ |
| ダッシュボード応答性 | チャート描画のレイテンシ | p95 < 2秒 |
| ユーザー継続率 | 週次アクティブ率 | 60%+ (導入2週間後) |
| PMツール対応数 | 統合済みインテグレーション | MVP: 2（ClickUp + Jira） |

## スコープ外

- PMツール上のタスク操作（作成・編集・削除）— 読み取り専用
- カスタムフィールドの完全対応 — MVPではステータスベースのメトリクスに集中
- チームメンバー管理・ロールベースアクセス制御 — MVPではシングルユーザー
- モバイルアプリ — Webダッシュボードのみ（レスポンシブ対応）
- リアルタイム通知・アラート機能
- AIによる予測・推奨機能

## ユーザーストーリー

- **スクラムマスター**として、複数プロジェクトのバーンダウンチャートを一画面で確認したい。なぜなら、スプリントの進捗を横断的に把握して早期にリスクを検知できるから。
- **エンジニアリングマネージャー**として、チームのベロシティトレンドを過去スプリントと比較したい。なぜなら、キャパシティプランニングの精度を上げたいから。
- **テックリード**として、タスクのリードタイム/サイクルタイムのボトルネックを特定したい。なぜなら、プロセス改善の根拠データが必要だから。
- **プロダクトオーナー**として、累積フローダイアグラムでWIP制限の遵守状況を確認したい。なぜなら、フロー効率を維持してデリバリーの予測可能性を高めたいから。
- **フリーランス開発者**として、無料でClickUpのスプリントレポートに相当する機能を使いたい。なぜなら、Businessプランに課金せずにアジャイルメトリクスを活用したいから。

## 技術要件

### 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フロントエンド | Next.js (TypeScript) | SSR/SSG対応、App Router、React Server Components |
| UIコンポーネント | shadcn/ui + Tailwind CSS | カスタマイズ性と軽量性 |
| チャート | Recharts | React統合、レスポンシブ、豊富なチャートタイプ |
| バックエンド | Go | 高並行処理、API連携の効率性、低メモリ消費 |
| データベース | PostgreSQL | JSON型サポート、時系列クエリ、信頼性 |
| ORM | sqlc | 型安全なSQLクエリ生成 |
| 認証 | NextAuth.js | OAuth統合、セッション管理 |
| インフラ | GCP Cloud Run | コンテナベース、オートスケール、従量課金 |

### アーキテクチャ

```
                 ┌──────────────────┐
                 │   Next.js (SSR)  │
                 │   Dashboard UI   │
                 └────────┬─────────┘
                          │ REST API
                 ┌────────▼─────────┐
                 │    Go API Server  │
                 │  ┌─────────────┐ │
                 │  │  Adapter層   │ │
                 │  │ ClickUp│Jira│ │
                 │  └─────────────┘ │
                 │  ┌─────────────┐ │
                 │  │ Metrics計算  │ │
                 │  │ Engine      │ │
                 │  └─────────────┘ │
                 └────────┬─────────┘
                          │
              ┌───────────▼───────────┐
              │     PostgreSQL        │
              │  tasks│snapshots│     │
              │  metrics│connections  │
              └───────────────────────┘
```

**Adapter パターン**: 各PMツールのAPI差分を吸収する共通インターフェースを定義。新規PMツール追加時はAdapterの実装のみで対応可能。

### 非機能要件

- **パフォーマンス**: ダッシュボード初期表示 < 3秒、チャート更新 < 1秒
- **セキュリティ**: APIキーはAES-256で暗号化保存、HTTPS必須、CORS制限
- **スケーラビリティ**: 1,000同時ユーザー対応（Cloud Run水平スケール）
- **可用性**: 99.5% uptime（Cloud Run SLA準拠）
- **データ保持**: スナップショットは最大1年間保持

## 競合分析

| 競合 | 価格帯 | 対応ツール | ギャップ |
|------|--------|-----------|---------|
| Screenful | $49-249/月 | ClickUp, Jira, Trello, Asana | 高価格。無料プランなし |
| Nave | $25/月～ | Jira, Azure DevOps | フロー分析特化。ClickUp非対応 |
| Hatica | エンタープライズ | Jira, GitHub, GitLab | 大規模チーム向け。個人/小チーム不向き |
| Velocity (Atlassian) | Jira内蔵 | Jiraのみ | Jira限定。他ツール非対応 |
| LinearのInsights | Linear内蔵 | Linearのみ | Linear限定 |

**差別化ポイント**: PMツール非依存の統一ダッシュボード。無料で始められ、複数PMツール横断でメトリクスを可視化・比較分析できる唯一のツール。

## 技術リスク

| リスク | 発生可能性 | 影響 | 軽減策 |
|--------|----------|------|--------|
| PMツールAPIのレート制限 | 高 | データ同期遅延 | Webhook併用、インクリメンタル同期、キャッシュ戦略 |
| タスク履歴APIの制限/不在 | 高 | バーンダウン精度低下 | Webhook+自前スナップショット蓄積、Time in Status API活用 |
| PMツールのAPI仕様変更 | 中 | アダプター破損 | バージョン管理、APIレスポンスのスキーマ検証 |
| PMツール利用規約の制限 | 中 | サービス停止リスク | 「分析ツール」としてのポジショニング、Partner Program参加 |
| スナップショットデータ量の増大 | 中 | ストレージコスト増 | パーティショニング、古いデータのアーカイブ/集約 |
