# ADR 004: MVP段階でのデモデータ使用

## ステータス
承認済み

## コンテキスト
MVP実装では、以下の2つのアプローチがある:
1. DB統合を先に行い、実データでチャートを表示する
2. デモデータでUI/チャートを先に動かし、DB統合は後続フェーズで行う

## 決定

**MVPではデモデータを使用し、DB統合は Phase 1 で行う。**

### 理由
- チャートUIの動作確認・UXフィードバックを先に得られる
- sqlcクエリ定義とマイグレーションSQLは作成済み（DB統合の準備は完了）
- DB統合にはテストコンテナ環境が必要で、MVPスコープを超える

### 未配線の成果物（Phase 1 で接続予定）
- `backend/sqlc/` — sqlcクエリ定義（コード生成 `sqlc generate` 未実行）
- `backend/migrations/001_initial_schema.sql` — DBマイグレーション（docker-entrypoint-initdb.d に配置済みだが、ハンドラーから未参照）
- `backend/internal/repository/` — リポジトリ層（空ディレクトリ、Phase 1 で実装）

## 影響
- MVP段階ではバックエンドAPIがデモデータを返す
- 実際のPMツール連携は接続テストエンドポイントのみ動作
- Phase 1 でDB統合時にハンドラーのデモデータをリポジトリ層に差し替え
