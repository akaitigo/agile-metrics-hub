# ADR 003: データ同期戦略

## ステータス
承認済み

## コンテキスト
PMツールからタスクデータを取得してローカルDBに保存する際の同期戦略を決める。
ClickUpはタスク履歴API（changelog）を公開していないため、全PMツール共通の方法が必要。

## 決定

### Webhook + ポーリングのハイブリッド方式

1. **初回同期**: APIでタスク全件取得 → DB一括保存
2. **増分同期（ポーリング）**: `updated_after` パラメータで差分取得（5分間隔）
3. **リアルタイム同期（Webhook）**: ステータス変更イベントをリアルタイムキャプチャ
4. **日次スナップショット**: 毎日0時にタスク状態のスナップショットを保存

### なぜハイブリッドか
- **Webhookのみ**: サーバー再起動・Webhook失敗時にデータ欠損
- **ポーリングのみ**: リアルタイム性が低い、レートリミット圧迫
- **ハイブリッド**: Webhookでリアルタイム性を確保しつつ、ポーリングで整合性を補完

### スナップショット方式を採用する理由
- ClickUpはタスク履歴APIが存在しない
- スナップショットなら過去の状態を再構築できる（バーンダウン/累積フロー用）
- 導入前のデータは取得不可 → ユーザーに「導入後から精度が向上する」旨を伝える

## 影響
- ストレージ: スナップショットが日次で増加 → パーティショニングで対応
- 初回同期: 大量タスクの場合はレートリミット制約（ClickUp: 100req/min）
- 導入直後: バーンダウンが不完全 → UX上の説明が必要
