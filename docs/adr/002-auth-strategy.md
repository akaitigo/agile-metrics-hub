# ADR 002: 認証方式の選定

## ステータス
承認済み

## コンテキスト
ユーザーのPMツールAPIキーを安全に管理し、アプリケーションへのアクセスを制御する方法を決める必要がある。

## 決定

### MVPフェーズ: APIキー直接入力
- ユーザーがPMツール（ClickUp/Jira）のAPIキーを直接入力
- サーバー側でAES-256-GCMで暗号化して保存
- MVPではアプリケーション自体の認証は省略（シングルユーザー前提）

### 理由
- OAuth2フローの実装はMVP段階ではオーバーエンジニアリング
- ClickUpのPersonal API Token、JiraのAPI Tokenはいずれもユーザーが簡単に発行できる
- セキュリティは暗号化保存 + HTTPS + CORS制限で担保

### 将来（Phase 1以降）
- NextAuth.jsでアプリケーション認証を追加
- ClickUp OAuth2 / Atlassian OAuth 2.0 (3LO) に移行
- マルチユーザー・チーム共有に対応

## 影響
- MVPはシングルユーザー限定
- APIキーのスコープ管理はユーザーの責任
- HTTPS必須（APIキーの平文通信を防止）
